## 学习建议

1. **不要跳阶段**。第一、二阶段是地基，直接看 Sentinel/Cluster 会迷失方向。
2. **动手验证**。每看完一篇，在本地 Redis 中复现文档里的场景（例如手动触发 BGSAVE、构造缓存击穿）。
3. **关联阅读**。比如看完「持久化」再看「主从复制」，会发现全量同步就是依靠 RDB 完成的——知识就串起来了。
4. **优先级**：如果时间有限，至少完成第一、二阶段（文档 1-16），这已经覆盖了 80% 日常开发所需的 Redis 知识。
